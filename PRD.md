# 产品需求文档 (PRD)
## AI驱动的移动自动化平台

---

## 1. 产品概述

### 1.1 产品名称
**FlowX** (Flow eXtended) - AI驱动的跨平台自动化引擎

### 1.2 产品定位
**下一代智能自动化平台**，提供两种输入模式：

**AI自然语言驱动**和**编程脚本驱动**（Python/JS），底层由高性能Rust引擎统一执行。

**一句话描述**：
> 说出你的需求或写下你的代码，Rust引擎+AI大脑让手机和电脑自动执行。

### 1.3 双模式输入

**模式1：AI自然语言驱动** 🤖
```
用户输入：打开淘宝，搜索笔记本电脑，截图前3个商品
AI理解 → 生成执行计划 → Rust引擎执行
```

**模式2：编程脚本驱动** 💻
```python
# Python脚本
from flowx import Device
device = Device.android()
device.open_app("com.taobao.taobao")
device.find_element(text="搜索").click()
device.input_text("笔记本电脑")
# ... Rust底层执行
```

**两种模式特点**：
| 特性 | AI驱动 | 脚本驱动 |
|------|--------|---------|
| **适用人群** | 所有人（零门槛） | 开发者、进阶用户 |
| **灵活性** | 中（AI理解能力限制） | 高（完全可控） |
| **学习成本** | 无 | 需学Python/JS |
| **复杂逻辑** | 有限（依赖AI能力） | 无限（任意编程） |
| **执行引擎** | Rust | Rust |
| **互操作** | AI可生成脚本 | 脚本可调用AI |

### 1.3 核心差异化
| 维度 | FlowX | AutoJS | Appium/Airtest |
|------|-------|--------|----------------|
| **脚本语言** | Python（主）+ JS（兼容） | JavaScript | Python |
| **AI能力** | ✅ 多模态理解、自然语言操作 | ❌ | ❌ |
| **性能** | Rust核心，高性能 | JavaScript，中等 | Python，较慢 |
| **学习曲线** | 低（AI辅助+Python） | 中（需学JS+Android） | 高（需专业知识） |
| **跨平台** | Android/iOS/Desktop | 仅Android | 主要Android |
| **商业模式** | 免费版+Pro订阅+企业版 | 一次性付费 | 开源免费 |

## 4. 功能需求

### 4.1 MVP功能（Phase 1，3个月）

**优先级：P0（必须有）**

#### 4.1.1 Python脚本引擎（重点培养）
```python
from flowx import Device

# 基础操作
device = Device.android()
device.click(100, 200)
device.swipe(100, 200, 300, 400)
device.input_text("Hello")

# 元素定位（Pythonic API）
button = device.find(text="确认")
button.click()

# 链式调用
device.find(text="设置").parent().find(id="icon").click()

# 截图和OCR
screen = device.screenshot()
texts = device.ocr(screen)
print(texts)

# 上下文管理器
with device.find(text="搜索框") as search:
    search.input("笔记本电脑")
    search.parent().find(text="搜索").click()
```

**核心能力**：
- Android设备连接（ADB）
- 基础操作：点击、滑动、输入
- 元素定位：文本、ID、xpath、图像
- 截图和OCR
- **Pythonic设计**：链式调用、上下文管理器、异常处理

**为什么Python是主推？**
1. AI时代的主流语言（数据分析、机器学习）
2. 语法简洁，学习曲线低
3. 生态丰富（pandas、requests等可直接用）
4. 用户画像匹配：技术人员更熟悉Python

#### 4.1.2 AI自然语言操作（核心差异化）
```python
from flowx import AI

ai = AI()
ai.execute("打开淘宝，搜索'笔记本电脑'，截图前3个商品")
```

**能力**：
- 自然语言→操作序列
- 屏幕理解（识别界面元素）
- 简单的任务规划

#### 4.1.3 AutoJS兼容层（获客手段）
```javascript
// legacy.js - 兼容AutoJS语法
auto.waitFor();
app.launch("com.tencent.mm");
let btn = text("发现").findOne();
btn.click();
```

**能力**：
- 运行80%的AutoJS脚本（核心API）
- 提供迁移工具：`flowx convert script.js` → 自动生成Python代码
- 迁移文档和教程

**定位说明**：
- ✅ **兼容是为了降低迁移成本**：让AutoJS用户无缝过渡
- ✅ **但不是核心竞争力**：我们不做"更好的AutoJS"
- ✅ **引导用户转向Python**：
  - 文档优先展示Python示例
  - 新特性只在Python提供
  - 转换工具推荐Python
  - 社区氛围培养Python文化

**迁移路径设计**：
```
AutoJS用户 → 用JS兼容层快速上手 → 看到AI功能（只有Python有）→ 
学习Python API → 习惯Python → 成为Python用户
```

#### 4.1.4 CLI工具
```bash

flowx run script.py          # 运行Python脚本
flowx run legacy.js          # 运行JS脚本（兼容模式）
```

## 5. 技术架构

### 5.1 整体架构（双模式输入 + Rust统一引擎）

```
┌──────────────────────┐  ┌──────────────────────┐
│   模式1: AI驱动       │  │  模式2: 脚本驱动      │
│                      │  │                      │
│  自然语言输入         │  │  Python脚本          │
│  "打开微信..."       │  │  device.click()      │
│       ↓              │  │       ↓              │
│  AI理解层            │  │  JavaScript脚本      │
│  (GLM-4V多模态)      │  │  text().click()      │
│       ↓              │  │       ↓              │
│  意图→操作序列        │  │  脚本解析器          │
└──────────────────────┘  └──────────────────────┘
         ↓                          ↓
         └──────────┬───────────────┘
                    ↓
    ┌───────────────────────────────────────┐
    │      Rust统一执行引擎（核心）          │
    │  ┌─────────────────────────────────┐  │
    │  │  指令层 (Unified Command Layer)  │  │
    │  │  - click(x,y)                    │  │
    │  │  - find_element(selector)        │  │
    │  │  - swipe(...)                    │  │
    │  └─────────────────────────────────┘  │
    │  ┌─────────────────────────────────┐  │
    │  │  设备控制                        │  │
    │  │  - UI自动化 (Accessibility)      │  │
    │  │  - 触摸/按键注入                 │  │
    │  │  - 截图/录屏                     │  │
    │  └─────────────────────────────────┘  │
    │  ┌─────────────────────────────────┐  │
    │  │  视觉能力                        │  │
    │  │  - OCR (PaddleOCR)               │  │
    │  │  - 模板匹配                      │  │
    │  │  - 图像识别                      │  │
    │  └─────────────────────────────────┘  │
    │  ┌─────────────────────────────────┐  │
    │  │  脚本运行时                      │  │
    │  │  - Python FFI (PyO3)             │  │
    │  │  - JS引擎 (QuickJS)              │  │
    │  └─────────────────────────────────┘  │
    └───────────────────────────────────────┘
                    ↓
    ┌───────────────────────────────────────┐
    │      平台抽象层 (Platform Layer)       │
    │  Android / iOS / Windows / macOS      │
    └───────────────────────────────────────┘
```

**架构核心理念**：
1. **Rust是唯一的执行引擎**：无论用户用AI还是写脚本，最终都翻译成Rust指令执行
2. **双输入通道**：AI和脚本是两种输入方式，不是两个引擎
3. **统一指令集**：`click()`, `swipe()`, `find_element()`等是底层原语，AI和脚本都调用它们
4. **性能保证**：所有性能关键路径都在Rust，Python/JS只是描述层

### 5.2 技术选型

| 模块 | 技术栈 | 理由 |
|------|--------|------|
| **核心引擎** | Rust | 性能、安全、跨平台编译 |
| **Python绑定** | PyO3 | 零开销FFI、类型安全 |
| **JS运行时** | QuickJS | 轻量（~600KB）、兼容AutoJS |
| **AI模型** | GLM-4V / Qwen-VL | 多模态、开源可控 |
| **图像处理** | OpenCV (Rust绑定) | 成熟、功能全 |
| **OCR** | PaddleOCR (Rust调用) | 中文支持好、精度高 |
| **Android** | JNI + Accessibility | 无需Root、官方API |
| **iOS** | libimobiledevice | 开源、跨平台 |
| **异步运行时** | tokio | Rust生态标准 |

### 10.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| AutoJS用户不愿迁移 | 获客失败 | 强化AI差异化、提供迁移工具 |
| 竞品模仿 | 优势缩小 | 快速迭代、建立社区护城河 |
| 政策监管 | 业务受限 | 合规化、避免灰产场景 |

### 10.3 运营风险

| 风险 | 影响 | 缓解措施 |
